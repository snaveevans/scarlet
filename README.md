# Scarlet v2

Scarlet is an autonomous coding system built around `agentloop`, a TypeScript CLI that:

1. Reads a PRD,
2. Generates and executes tasks,
3. Validates each change,
4. Performs self-review and reflection,
5. Persists reusable knowledge in `.scarlet/knowledge`.

## Quick start

### 1) Install

```bash
cd agentloop
pnpm install
pnpm build
npm link
```

### 2) Set your LLM API key

Use one provider:

```bash
export ANTHROPIC_API_KEY=your_key
# or
export OPENAI_API_KEY=your_key
```

### 3) Generate a PRD template

```bash
cd /path/to/target/project
agentloop init --format v2 --output ./prd.md
```

### 4) Fill the PRD and run

```bash
# Full run (v2 runs comprehension automatically)
agentloop run ./prd.md

# Or run comprehension only
agentloop comprehend ./prd.md
```

### 5) Inspect/resume

```bash
agentloop status
agentloop resume
```

## PRD format (v2)

V2 focuses on intent and acceptance criteria; tasks are generated by comprehension.

Minimal shape:

```markdown
# Product Requirements Document v2

## Name
My Feature

## Summary
Short problem statement and target behavior.

## Acceptance Criteria
- [ ] AC-1: User can do X
- [ ] AC-2: System handles Y

## Constraints
- Keep existing API contracts

## ADRs
### ADR-1: Use existing queue
- Decision: Reuse current queue worker
- Rationale: Avoid migration risk
```

Legacy v1 PRDs are still supported for direct task authoring.

## Configuration

Configuration precedence:
1. CLI flags
2. `.agentloop/config.json`
3. Built-in defaults

Example:

```json
{
  "agent": "scarlet",
  "maxAttempts": 3,
  "autoCommit": true,
  "skipFailedDeps": true,
  "validationSteps": ["typecheck", "lint", "test", "build"],
  "contextBudget": 12000,
  "taskTimeout": 600000,
  "validationTimeout": 60000,
  "llm": {
    "provider": "anthropic",
    "model": "claude-sonnet-4-5-20250929",
    "maxTokens": 8192,
    "temperature": 0
  }
}
```

### Model routing

`modelRouting` allows per-phase model selection (e.g. `explore`, `decompose`, `code`, `review`, `reflect`) and optional complexity-specific code routing (`low|medium|high`).

## Knowledge system

Scarlet stores learned patterns under `.scarlet/`:

- `.scarlet/knowledge/skills.json`
- `.scarlet/knowledge/pitfalls.json`
- `.scarlet/knowledge/tools/*.json`
- `.scarlet/context.md` (generated context snapshot for future runs)

Knowledge is updated during reflection after review-approved runs.

## Architecture overview

Execution flow:

1. **Comprehension** (`explore -> decompose -> validate plan`)
2. **Scaffolding** (create baseline files/tests)
3. **Task execution loop** (implement + validate + retries)
4. **Self-review** (LLM review against PRD + targeted fix tasks)
5. **Reflection** (extract skills/pitfalls/tools + refresh context)

Core modules live in `agentloop/src/`:

- `llm/` providers and routing
- `tools/` tool runtime exposed to the agent
- `comprehension/`, `scaffold/`, `executor/`, `review/`, `reflection/`
- `knowledge/` persistent learning store
- `memory/` layered context manager
- `state/` resumable run state and progress logging

See `docs/architecture.md` for the detailed architecture reference.
